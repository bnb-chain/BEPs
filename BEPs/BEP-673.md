## 1. Summary

This Meta BEP lists the BEPs formally Scheduled for Inclusion in the Pasteur network upgrade.

## 2. Specification

The Pasteur network upgrade comprises a total of 3 BSC only BEPs.

### BEPs Scheduled for Inclusion

#### BEPs need hard fork

* [BEP-667](./BEP-667.md) : Introduce Vote Interval to Relax Fast Finality Consensus Constraint
* [BEP-670](./BEP-670.md) : Short Block Interval Phase Four: 250ms

#### BEPs without hard fork
* [BEP-675](./BEP-675.md) : Builder-Proposed Block with Validator Blind Signing

## 3. Rationale

This Meta BEP provides a global view of all changes included in the Pasteur network upgrade, as well as links to full specification.

The three BEPs in this upgrade are closely related and together form a coherent set of changes centered on reducing the block interval from 450ms to 250ms:

- **[BEP-670](./BEP-670.md)** is the core change of this upgrade, reducing the block interval from 450ms to 250ms to lower transaction inclusion latency.

- **[BEP-667](./BEP-667.md)** is a necessary companion to BEP-670. As the block interval shortens, validators have less time to collect and broadcast votes, which risks destabilizing the fast finality mechanism. BEP-667 introduces a Vote Interval, hardcoded at hard fork time, that decouples the voting frequency from the block production frequency. With Vote Interval set to 2, validators cast a finality vote once every two blocks, providing sufficient time for vote propagation and collection while still maintaining sub-second finality.

- **[BEP-675](./BEP-675.md)** addresses a long-standing throughput gap on the builder path. Prior to this BEP, the time available for builder transaction packing was significantly shorter than for local mining, resulting in lower builder-path throughput. By redesigning the block proposal flow so that builders sign and submit the full block body while validators perform blind signing on the block header, the time available for builder transaction packing is extended by approximately 50%, substantially narrowing this gap and making the builder path viable at the 250ms block interval introduced by BEP-670.
